# Design Log Entry Template

Copy the block below and paste it at the bottom of `DESIGN-LOG.md`, below the `## Entries` heading. Fill in every field. Write `N/A` rather than leaving a field blank.

---

```markdown
### YYYY-MM-DD — <brief title>
<!-- Date of the decision/configuration. Title: action + subject, e.g. "Tool choice — Prompt node vs AI Builder" -->

**Decision / Configuration**
<!-- What was decided or configured. Use exact names, option labels, and values as they appear in the UI or docs.
     Do not paraphrase — copy field names and settings verbatim. -->

**Rationale**
<!-- Why this choice was made. Include any constraints that drove the decision:
     licensing, timeline, capability gaps, team skill, environment limitations, etc. -->

**Alternatives Considered**
<!-- List each alternative that was seriously considered and the specific reason it was rejected.
     Format: "- **Alternative name:** reason rejected." -->

**References**
<!-- Where to find this in the product or documentation. Include as many as apply:
     - Screen location / nav path (e.g. "Copilot Studio: Settings > Skills")
     - Microsoft Learn URL or doc title
     - Table name, solution component name, flow name
     - Repo file path or runbook step number -->

**Risks / Watch-outs**
<!-- Known risks, open questions, or conditions that could invalidate this decision.
     Examples: feature still in preview, environment-specific capability, dependency on another decision.
     Use N/A if there are genuinely no risks to note. -->
```
